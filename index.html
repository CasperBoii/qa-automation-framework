Engineering Rationale

Why it's built this way.

🔀
Why cross-model review?
A model shares the same blind spots as its own output. Using Gemini to review Claude's specs introduces a genuinely independent perspective.
🔍
Why ±15-line validation?
LLMs report approximate line numbers. The sub-agent reads source around each cited line and confirms the issue exists — filtering out hallucinations.
💰
How is free-tier quota managed?
SHA-256 hash of target folder plus shared fixtures. If nothing changed, the review is skipped — re-runs are free.
🚦
Why never auto-apply fixes?
AI reviewers are thorough but over-eager. A finding may be valid but out of scope. Every fix requires explicit approval.
🧬
How do you trust a green suite?
A green run only proves the tests passed — not that they'd catch a bug. So I mutation-test: break the code on purpose and confirm the test turns red. If it stays green, the assertion is wrong, not the app.

Bug Prioritization & Release Triage

Severity ≠ Priority.

A reusable checklist for ranking open defects before a release. Severity is a property of the bug itself; Priority is a business decision.

S
Severity Rubric
How bad the defect is when it happens
SeverityUse when
CriticalCorrupts data, breaks core flow with no workaround, money/pay/legal impact
MajorImportant function degraded, but a workaround or fallback exists
MinorLower-impact validation / UX issue; no data or flow risk
TrivialTypos, spacing, copy — zero functional impact
P
Priority Factors
Impact × likelihood, adjusted for context
  • Workaround? — A real alternative lowers priority. Verify it actually works.
  • Reach — % of users / platforms affected.
  • Data / money — Near-automatic must-fix.
  • Silent failure — More dangerous than a loud one. Weight it up.
!
Platform Fact · iOS WebKit
"Just use Chrome on iPhone" is NOT a workaround for a Safari bug.
iOS browsers all use WebKit — Chrome/Firefox on iOS render with the same engine. Exception: EU DMA (2024+) allows alternative engines.
✉
Communicate the Call
A ≤150-word note to PM / Dev Lead — blockers first, then one concrete ask.
Each bug gets a verdict + a one-line why. Calls that hinge on data I don't have are flagged as explicit questions; every deferral states what would flip it back to a blocker.
